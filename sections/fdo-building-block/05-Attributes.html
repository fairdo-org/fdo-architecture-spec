FDO Attributes are properties of an FDO
  • [R5-1] Each tuple in an FDO Record is called an FDO Attribute (short: attribute).
  • [R5-2] An attribute denotes a property of an FDO in form of a tuple that consists of the following two elements:
    • The first element of the tuple MUST be either a PID that resolves to an attribute definition, or a human-readable name of an attribute.
    • The second element (called value) can be any literal. If the first element refers to an attribute definition, then the value MUST be successfully validated (see section 8) against the attribute definition at least upon creation and update.

  • [R5-3] There are three qualitative categories of attributes (or attribute definitions): Mandatory, optional, and community-defined attributes.
    • The mandatory attributes are defined by the FDO Forum. These attributes MUST be used in each FDO record. This set is defined in the root profile (see section 5.4).
    • The optional attributes are defined by the FDO Forum. It is a list of pre-defined attributes which are frequently reused by use-cases within their FDO records and profiles (see Table X in appendix). Use-cases SHOULD reuse these attributes, whenever suitable, for better harmonization of FDO information.
    • Community-defined attributes are any attributes that are not defined by the FDO Forum. They appear in different forms, with decreasing level of machine-actionability in the following enumeration:
      • A Community attribute MAY be specified as an attribute definition, according to [R5-4] - [R5-15], against which the instantiated attribute in an FDO record MUST be validated. The attribute definition is instantiated as the following attribute:
        • first element: the PID or the name of the attribute definition
        • value: any literal validating against the first element
      • Community attributes MAY be used in records by referencing existing elements in vocabularies, terminologies or ontologies. Validation MAY be performed, but is not guaranteed. The attribute looks as follows:
        • first element: reference to element from vocabulary, terminology or ontology
        • value: instantiates the first element
        See section 5.7.5 for more details.
        • Community attributes MAY be used in records by a simple human-readable label. No validation can be performed here.
        • first element: human-readable label
        • value: any literal

      These levels do not impose additional requirements, but help repositories and users to assess validation behaviour, interoperability, and long-term interpretability of FDO Records.
